COS102: Problem‑Solving & Programming

structured learning · algorithms · C · Visual Basic 6

Modules at a glance

  • 1. Problem‑Solving techniques
  • 2. C programming language
  • 3. Visual Basic 6 (VB6)

Module 1: Introduction to Problem‑Solving Techniques

  • Core concepts: zero‑I.Q. computer & programmer responsibility, problem formulation, the 5‑stage program development cycle (analysis, design, coding, testing, documentation).
  • Algorithms: definition (complete, unambiguous, finite steps), choosing best algorithm (time requirement), steps in algorithm development, examples: average of three numbers, simple interest, area of triangle, largest of three, factorial, prime testing.
  • Flowcharts: ISO symbols – oval (terminal), parallelogram (I/O), rectangle (process), diamond (decision), flow lines, connectors (on‑page / off‑page). advantages: pictorial logic, easy checking, communication.
  • Pseudocode: abstract English‑like description, easy to read/modify. other tools: hierarchy charts (HIPO) for module relationships.

Module 2: Features of the ‘C’ Programming Language

  • Introduction to C: Dennis Ritchie, Bell Labs (early 1970s). features – simple, versatile, rich operators, fast execution, system programming. applications: UNIX, compilers, databases.
  • Fundamentals: character set, structure of a C program, data types (int, float, double, char), operations & expressions, assignment statement, variables & pointers (memory address).
  • Input / output: printf() function, statements end with ;.
  • Control flow: conditional (if, else, ladder, nested if-else – using ==), loops (while, do-while, for), break/continue, switch (multi‑branch), goto (unconditional).

Module 3: Introduction to Visual Basic 6 (VB6)

  • Core concepts: object‑based & event‑driven programming, GUI apps, rapid application development (RAD).
  • IDE components: menu bar, tool bar, tool box, properties window, project explorer, code window, form layout window, immediate window (debugging), object browser.
  • Controls & objects: Label, TextBox, CommandButton, OptionButton, CheckBox, Frame, ListBox, ComboBox, DriveListBox, DirListBox, FileListBox, Data Control, Timer. Form is an object but not a control.
  • Common properties: visual (BackColor, ForeColor, Font, Caption, Text, Visible), layout (Left, Top, Width, Height), behavioural (Enabled, MousePointer, TabIndex).
  • Programming basics: start Standard EXE, set properties (design‑time vs. run‑time), write event procedures (e.g., Command1_Click()), comments with apostrophe ('), naming conventions (txt for TextBox, cmd for CommandButton).
